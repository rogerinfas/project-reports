\subsection{Desarrollo de metodología}

En esta sección se describe la ejecución metodológica del proyecto en su fase operativa, detallando cómo se aplicó el enfoque Scrum adaptado durante la construcción progresiva de la propuesta tecnológica. El propósito es evidenciar, de manera ordenada, la relación entre planificación, desarrollo técnico y validación de resultados en cada iteración.

\subsubsection{Análisis de requisitos}

En esta etapa se identificó la problemática central del estudio y se delimitaron las necesidades funcionales y técnicas del sistema. El objetivo fue establecer una base clara para iniciar el desarrollo por Sprints.

\begin{itemize}
    \item Identificación de stakeholders.
    \item Levantamiento de información del problema.
    \item Definición de requerimientos funcionales.
    \item Definición de requerimientos no funcionales.
    \item Elaboración de historias de usuario.
    \item Definición de criterios de aceptación.
    \item Priorización inicial en el Product Backlog.
\end{itemize}

\subsubsection{Diseño del sistema}

La fase de diseño permitió transformar los requerimientos en una solución técnicamente estructurada, manteniendo coherencia con el enfoque de Clean Architecture y el modelado orientado al dominio.

\begin{itemize}
    \item Definición de la arquitectura del sistema.
    \item Modelado del dominio (DDD).
    \item Identificación de entidades y agregados.
    \item Diseño de casos de uso (capa de aplicación).
    \item Diseño de base de datos.
    \item Definición de interfaces y contratos (API).
    \item Diseño de la estructura del frontend.
\end{itemize}

\subsubsection{Planificación de iteraciones (Sprints)}

Esta fase organizó el trabajo incremental de acuerdo con Scrum adaptado, estableciendo objetivos semanales concretos y entregables verificables por iteración.

\begin{itemize}
    \item Organización del Product Backlog.
    \item Selección de historias de usuario por Sprint.
    \item Descomposición en tareas técnicas.
    \item Estimación de esfuerzo.
    \item Definición de entregables por iteración.
\end{itemize}

\subsubsection{Implementación y construcción incremental}

En esta etapa se desarrollaron los componentes funcionales del sistema, integrando backend, frontend y persistencia espacial de datos de forma progresiva.

\begin{itemize}
    \item Desarrollo de la lógica de negocio.
    \item Implementación de casos de uso.
    \item Desarrollo de APIs.
    \item Implementación de persistencia de datos.
    \item Desarrollo de la interfaz de usuario.
    \item Integración entre frontend y backend.
    \item Aplicación de principios SOLID y buenas prácticas.
\end{itemize}

\subsubsection{Validación y control de calidad}

La fase de validación aseguró que cada incremento cumpla con los criterios de aceptación definidos y que el sistema mantenga estabilidad funcional al cierre de cada Sprint.

\begin{itemize}
    \item Pruebas funcionales por iteración.
    \item Verificación de criterios de aceptación.
    \item Registro y corrección de incidencias.
    \item Ajustes finales de estabilidad y usabilidad.
\end{itemize}
